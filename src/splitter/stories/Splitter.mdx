<Subtitle>
	Компонент для интерактивного разделения рабочей области на две панели с поддержкой вложенности, клавиатуры и ограничений по размеру
</Subtitle>

`Splitter` подходит для интерфейсов, где пользователь должен самостоятельно перераспределять пространство между соседними областями: навигацией и контентом, таблицей и деталями, превью и журналом событий.

Компонент ожидает ровно два прямых дочерних элемента. Если нужно больше областей, их лучше собирать через вложенные `Splitter`, чтобы сохранить простую модель поведения и предсказуемый resize.

## Основной сценарий

<Primary />

```tsx
import { Splitter } from "@/shared/ui";

export function Example() {
	return (
		<div style={{ height: 420 }}>
			<Splitter initial={0.45}>
				<div>Левая панель</div>
				<div>Правая панель</div>
			</Splitter>
		</div>
	);
}
```

<Controls of={SplitterStories.Basic} />

## Что важно про поведение

- Во время перетаскивания компонент обновляет размеры панелей напрямую через DOM, а состояние React фиксирует только после завершения drag. Это уменьшает число лишних перерендеров и делает resize заметно плавнее.
- Значения `initial`, `min` и `max` автоматически нормализуются в диапазон `0..1`.
- Если `min` и `max` заданы в обратном порядке, компонент сам исправляет диапазон.
- Разделитель управляется клавиатурой: стрелки двигают границу пошагово, `Home` переводит в минимум, `End` в максимум.

## Горизонтальное разделение

Горизонтальный режим полезен для layout вида "контент сверху, журнал или детали снизу".

<Canvas of={SplitterStories.Horizontal} />

```tsx
import { Splitter } from "@/shared/ui";

export function HorizontalLayout() {
	return (
		<div style={{ height: 420 }}>
			<Splitter direction="horizontal" initial={0.55}>
				<div>Верхняя панель</div>
				<div>Нижняя панель</div>
			</Splitter>
		</div>
	);
}
```

## Разделение на 4 части

Чтобы собрать четыре независимые области, используйте `Splitter` внутри `Splitter`. Такой подход даёт предсказуемую вложенную структуру и хорошо масштабируется для сложных рабочих экранов.

<Canvas of={SplitterStories.NestedFourPanels} />

```tsx
import { Splitter } from "@/shared/ui";

export function FourPaneLayout() {
	return (
		<div style={{ height: 420 }}>
			<Splitter initial={0.48} min={0.25} max={0.75}>
				<Splitter direction="horizontal" initial={0.58}>
					<div>Левая верхняя панель</div>
					<div>Левая нижняя панель</div>
				</Splitter>
				<Splitter direction="horizontal" initial={0.36}>
					<div>Правая верхняя панель</div>
					<div>Правая нижняя панель</div>
				</Splitter>
			</Splitter>
		</div>
	);
}
```

## Ограничения и клавиатура

В этом сценарии удобно проверять, что пользователь не может схлопнуть область сильнее допустимого минимума, а разделитель стабильно работает не только мышью, но и с клавиатуры.

## Практические рекомендации

- Родительский контейнер должен иметь явную высоту, иначе `Splitter` не сможет корректно распределить пространство по вертикали.
- Держите в `Splitter` только два прямых потомка. Для трёх и более областей используйте вложенность.
- Если в панелях живут тяжёлые таблицы, графики или сложные формы, лучше избегать лишней бизнес-логики, завязанной на каждое движение разделителя.
- Для touch-сценариев разделитель уже блокирует нативный скролл жеста на самом handle, поэтому drag остаётся предсказуемым.
